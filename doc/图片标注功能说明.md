# 答题卡图片标注功能说明

## 功能概述

在客观题阅卷完成后，系统会自动生成以下三种图片文件并保存到 `read` 文件夹：

1. **标注图片** (`*_annotated.jpg`) - 在原始答题卡上标注错题号和总分
2. **原始备份** (`*_original.jpg`) - 原始答题卡的备份
3. **评分摘要** (`*_summary.jpg`) - 详细的评分信息摘要图片

## 文件命名规则

- 如果成功识别姓名和学号：`姓名_学号_类型.jpg`
- 如果未识别姓名学号：`原文件名_类型.jpg`

## 标注内容

### 标注图片 (*_annotated.jpg)
- **总分信息**：显示在图片顶部，蓝色边框
- **错题信息**：显示错误题号，红色边框
- **空白题信息**：显示未作答题号，橙色边框
- 自动分行显示，避免文字过长

### 评分摘要 (*_summary.jpg)
- 学生基本信息（姓名、学号）
- 详细分数信息（总分、客观题分、主观题分）
- 完整的错题列表
- 完整的空白题列表

## 使用方式

该功能已集成到现有的阅卷流程中，无需额外操作：

1. 运行批量阅卷脚本
2. 客观题阅卷完成后自动生成标注图片
3. 查看 `read` 文件夹中的结果文件

## 技术实现

- 使用 OpenCV 进行图片处理和文字绘制
- 自动创建 `read` 输出目录
- 支持中文文件名（自动处理编码问题）
- 错误处理和日志记录

## 注意事项

1. 确保原始图片路径正确传递
2. `read` 文件夹会自动创建
3. 如果图片标注失败，不会影响正常的阅卷流程
4. 所有标注过程都会记录在识别日志中

## 示例输出

```
read/
├── 张三_2023001_annotated.jpg    # 标注后的答题卡
├── 张三_2023001_original.jpg     # 原始答题卡备份  
└── 张三_2023001_summary.jpg      # 评分摘要图片
```

这样可以方便地查看每张答题卡的错误情况，便于后续分析和复查。
