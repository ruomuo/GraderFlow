# 矩形填涂框识别改进方案

## 问题描述

原有的答题卡识别系统在处理矩形填涂框时存在以下问题：
1. **填涂检测不准确**：矩形填涂框的面积填涂或未填涂都一样，简单的像素计数方法无法准确判断
2. **Top N筛选问题**：在多选题中，固定的Top N限制会导致某些选项被遗漏

## 解决方案

### 1. 改进的矩形填涂检测算法

新增 `detect_rectangle_filling()` 函数，采用多种检测方法综合判断：

#### 方法1：边框去除 + 内部区域分析
- 动态计算边框宽度：`max(2, min(w//10, h//10))`
- 去除矩形边框，只分析内部区域的填涂密度
- 阈值：内部填涂密度 > 0.3

#### 方法2：连通域分析
- 使用 `cv2.connectedComponentsWithStats()` 分析连通域
- 计算最大连通域面积占比
- 阈值：最大连通域占比 > 0.2

#### 方法3：中心区域密度分析
- 重点分析矩形中心区域（1/4面积）的填涂情况
- 阈值：中心区域填涂密度 > 0.4

#### 方法4：边缘对比分析
- 比较内部区域和边缘区域的像素密度差异
- 阈值：内部密度 > 边缘密度 × 1.5

#### 综合判断策略
- 至少满足2个条件才认为是填涂
- 提供详细的调试信息便于优化

### 2. 改进的轮廓筛选策略

#### 取消固定Top N限制
- 不再使用堆排序筛选固定数量的轮廓
- 改为基于特征的动态筛选

#### 基本几何特征筛选
```python
if (0.6 < aspect_ratio < 2.5 and 
    area > 500 and  # 最小面积阈值
    w > 10 and h > 10):  # 最小尺寸阈值
```

#### 智能面积筛选
- 当轮廓数量过多时，计算面积的中位数和标准差
- 保留面积在中位数±2倍标准差范围内的轮廓
- 过滤异常大小的轮廓

#### 位置排序优化
- 按位置排序：`(y坐标, x坐标)` 从上到下，从左到右

### 3. 参数优化

#### Top N值调整
- 从默认的5增加到20，减少多选题选项遗漏

#### 宽高比范围扩大
- 从 `0.8-1.8` 调整为 `0.6-2.5`，更适合矩形填涂框

## 代码修改点

### 1. 新增函数
- `detect_rectangle_filling(roi, debug=False)`: 改进的矩形填涂检测算法

### 2. 修改的函数
- `recognize_answer_sheet()`: 
  - 默认top_n从5改为20
  - 替换轮廓筛选逻辑
  - 使用新的填涂检测算法

### 3. 关键改进
```python
# 旧方法
filled = cv2.countNonZero(option_roi) / (w * h) > 0.6

# 新方法
filled = detect_rectangle_filling(option_roi, debug=False)
```
